# 📱🖥️ Cross-Platform Testing Guide - ПСИХОЛОГІЯ СИЛИ

## 🎯 Ваша тестова конфігурація
- **🖥️ Ноутбук:** Windows 10/11 з Chrome/Edge
- **📱 Телефон:** iPhone 12 з Safari
- **🌐 Мережа:** Локальна мережа для обох пристроїв

## 🚀 Швидкий старт

### 1. Запуск на Windows ноутбуці
```bash
# Відкрийте CMD/PowerShell
cd C:\Users\user\CascadeProjects\psychology-of-power
npm install
npm run dev -- --host
```

### 2. Отримання IP адреси
```bash
# В терміналі виконайте:
ipconfig
# Знайдіть "IPv4 Address" (наприклад: 192.168.1.100)
```

### 3. Підключення iPhone 12
```
1. Підключіть iPhone до тієї ж Wi-Fi мережі
2. Відкрийте Safari на iPhone
3. Перейдіть на http://192.168.1.100:3000
4. Натисніть "Поділитися" → "На домашній екран"
```

## ✅ Перевірка синхронізації даних

### 🔄 Тестування синхронізації
1. **Створіть інцидент на Windows**
2. **Перевірте на iPhone** - дані мають бути доступні офлайн
3. **Створіть інцидент на iPhone**
4. **Перевірте на Windows** - синхронізація через IndexedDB

### 📊 Моніторинг даних
```javascript
// В консолі браузера на обох пристроях
localStorage.clear();
indexedDB.deleteDatabase('PsychologyOfPowerDB');
// Перезавантажте додаток для чистого тесту
```

## 🔄 Тестування синхронізації між пристроями

### Сценарій 1: Інцидент створено на Windows
1. **Windows:** Створіть інцидент з відео
2. **Windows:** Перевірте збереження в IndexedDB
3. **iPhone:** Відкрийте додаток
4. **iPhone:** Перевірте доступність даних офлайн

### Сценарій 2: Інцидент створено на iPhone
1. **iPhone:** Створіть інцидент через екстрений режим
2. **iPhone:** Перевірте збереження
3. **Windows:** Оновіть сторінку
4. **Windows:** Перевірте наявність нового інциденту

### Сценарій 3: Екстрений режим на обох пристроях
1. **iPhone:** Активуйте SOS режим
2. **Windows:** Одночасно активуйте SOS режим
3. **Обидва:** Перевірте якість відео та геолокацію
4. **Синхронізація:** Перевірте обмін даними

## 🔧 Налаштування локальної мережі

### Windows Firewall
```bash
# Дозволіть порт 3000 в Windows Firewall
Windows Defender Firewall → Advanced Settings → Inbound Rules → New Rule
Port: 3000, Protocol: TCP, Action: Allow
```

### Router налаштування
```
Переконайтесь що обидва пристрої в одній мережі:

- iPhone: 192.168.1.x
```

## 🚨 Відомі проблеми та рішення

### Проблема: iPhone не бачить сервер
**Рішення:**
```bash
# Перевірте IP адресу
ipconfig
# Переконайтесь що порт 3000 відкритий
netstat -an | findstr :3000
```

### Проблема: Камера не працює на iPhone
**Рішення:**
```
iPhone → Settings → Safari → Camera → Allow
iPhone → Settings → Privacy → Camera → Safari
```

### Проблема: Геолокація неточна
**Рішення:**
```
iPhone → Settings → Privacy → Location Services → Safari → While Using
Windows → Settings → Privacy → Location → Allow apps to access
```
